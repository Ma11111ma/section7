## API 仕様書 (MVP 版)

フロントエンドとバックエンド間の唯一の通信ルールを定義します。

### エンドポイント: `POST /api/weight`

今日の体重を記録し、それに基づいて擬似入金処理を行い、最新の口座残高を返します。

---

#### リクエスト (Request)

フロントエンドからバックエンドへ送信するデータです。

- **ヘッダー (Headers)**

  - `Content-Type: application/json`

- **ボディ (Body)**

| キー     | 型       | 必須 | 説明                                      |
| :------- | :------- | :--- | :---------------------------------------- |
| `weight` | `Number` | ✅   | ユーザーが入力した今日の体重 (例: `75.5`) |

- **リクエストボディの例:**
  ```json
  {
    "weight": 75.5
  }
  ```

---

#### レスポンス (Response)

バックエンドからフロントエンドへ返却するデータです。

- **成功時 (ステータスコード: `200 OK`)**

| キー             | 型       | 説明                                                 |
| :--------------- | :------- | :--------------------------------------------------- |
| `currentBalance` | `Number` | GMO あおぞら NET 銀行 API から取得した最新の口座残高 |
| `message`        | `String` | 処理成功を示すメッセージ                             |

- **成功レスポンスの例:**

  ```json
  {
    "currentBalance": 120000,
    "message": "体重を記録し、残高を更新しました！"
  }
  ```

- **失敗時 (ステータスコード: `400 Bad Request`など)**

| キー    | 型       | 説明                                   |
| :------ | :------- | :------------------------------------- |
| `error` | `String` | 何が問題だったかを示すエラーメッセージ |

- **失敗レスポンスの例:**
  ```json
  {
    "error": "体重は有効な数値で入力してください。"
  }
  ```
