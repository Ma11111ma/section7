## 【デモ会発表】体重・残高連携アプリ（要点まとめ）

### 1. コンセプト

- **課題**: 健康管理（ダイエットなど）のモチベーション維持の難しさ。
- **解決策**: 体重記録と銀行口座を連携させ、日々の頑張りを「資産の可視化」という形で実感できるアプリ。

---

### 2. 主な機能（デモの流れ）

- **初期表示**:
  - 仮の残高（300,000 円）と体重（80.0kg）を表示。
- **体重記録**:
  - ユーザーが新しい体重を入力し、「記録する」ボタンを押す。
- **結果表示（動的更新）**:
  - **体重**: 入力値に更新。
  - **残高**: 銀行 API をリアルタイムで呼び出し、実際の残高に更新。
  - **メッセージ**: 前回との差分（体重の増減、残高の増加）を表示。
  - **キャラクター**: 体重が減ると画像が変化。

---

### 3. 技術的なハイライト

- **フルスタック開発**:
  - **フロントエンド**: Next.js (React)
  - **バックエンド**: Next.js API Routes
- **外部 API 連携**:
  - GMO あおぞらネット銀行の Sunabar API（残高照会）と連携。
  - API の仕様解釈とデバッグを乗り越え、安定した通信を実装。
- **データベース**:
  - **DB**: PostgreSQL
  - **ORM**: Prisma
  - 記録された体重データを永続化。

---

### 4. 今後の展望

- **機能拡張案**:
  - ご褒美の自動振込機能の実装。
  - 体重・残高推移のグラフでのデータ可視化。
  - 複数ユーザーに対応するためのユーザー認証機能。

---

### 5. まとめ

- 3 日間で、健康管理とリアルな金融データを連携させるフルスタックアプリケーションのプロトタイプを完成させた。
