## 練習タスク：具体的なコマンド手順

以下に、「誰が」「どのブランチで」「どのファイルを」「どのコマンドで」操作するのか、A さんと B さんの対話形式で手順を追っていきます。

### 最初のタスク (担当：A さん、レビュー：B さん)

#### 【A さんの操作 🧑‍💻】

1.  **Issue 作成 (ブラウザ)**

    - GitHub の`practice`リポジトリで`Issues`タブを開き、`New issue`をクリック。
    - **Title:** `README.mdの基本的な枠組みを作成する` と入力。
    - **Assignees:** 自分（A さん）をアサイン。
    - `Submit new issue`をクリック。（Issue 番号が`#1`になったとします）

2.  **Project ボードへの追加と移動 (ブラウザ)**

    - `Projects`タブを開き、`ToDo`カラムに今作った Issue (`#1`) を追加します。
    - 作業を始めるので、`#1`のカードを`ToDo`から`In Progress`へドラッグ＆ドロップします。

3.  **ローカルでの作業準備 (ターミナル)**

    - **誰が:** A さん
    - **どのブランチで:** まずは`main`ブランチを最新の状態にします。
    - **コマンド:**

      ```bash
      # mainブランチに切り替える
      git switch main

      # GitHub上の最新の状態をローカルに持ってくる
      git pull origin main
      ```

    - 次に、今回の作業専用のブランチを作成します。
    - **誰が:** A さん
    - **どのブランチに:** `main`ブランチから、Issue 番号`1`に対応する新しいブランチを作成します。
    - **コマンド:**
      ```bash
      # Issue番号1番の作業をするためのブランチを作成し、そこに移動する
      git switch -c 1-create-readme
      ```

4.  **ファイルの作成と編集 (エディタ)**

    - **誰が:** A さん
    - **どのブランチで:** `1-create-readme`
    - **どのファイルを:** `README.md`（新規作成）
    - VSCode などのエディタで`README.md`ファイルを作成し、以下の内容を記述して保存します。

      ```markdown
      # Money Fit App 練習リポジトリ

      このリポジトリは、GitHub の Issue, Pull Request, Project 機能の練習用です。
      ```

5.  **コミットとプッシュ (ターミナル)**

    - **誰が:** A さん
    - **どのブランチで:** `1-create-readme`
    - **どのファイルを:** `README.md`
    - **コマンド:**

      ```bash
      # 変更内容をステージングする
      git add README.md

      # コミットメッセージを付けてコミットする
      git commit -m "feat: Create initial README.md"

      # GitHubに作業ブランチをプッシュする
      git push origin 1-create-readme
      ```

6.  **プルリクエスト作成 (ブラウザ)**
    - GitHub リポジトリのページに行くと、「`1-create-readme` had recent pushes」と表示されるので、`Compare & pull request`ボタンを押します。
    - **Title:** `README.mdの基本的な枠組みを作成` (自動で入っているはずです)
    - **Description:** 本文にマジックキーワード `closes #1` と入力します。
    - **Reviewers:** B さんをレビュアーに指定します。
    - `Create pull request`をクリックします。

---

### 【B さんの操作 🧑‍🔬】

1.  **レビュー (ブラウザ)**

    - GitHub で A さんからのレビュー依頼を確認します。
    - プルリクエストの`Files changed`タブを開き、変更内容に問題がないか確認します。
    - 問題なければ`Review changes`ボタンを押し、`Approve`を選択して`Submit review`します。

2.  **マージ (ブラウザ)**

    - `Merge pull request`ボタンを押し、確認画面でもう一度ボタンを押して`main`ブランチに変更を取り込みます。
    - マージ後、不要になった作業ブランチを削除する`Delete branch`ボタンが表示されるので、クリックして削除します。

3.  **最終確認 (ブラウザ)**
    - `Issues`タブを開き、`#1`が自動で`Closed`になっていることを確認します。
    - `Projects`タブを開き、`#1`のカードが自動で`Done`カラムに移動していることを確認します。
